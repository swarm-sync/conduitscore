# DarkMirror — ConduitScore Free/Paid Gate Brainstorm
# 20 Worst Possible Ideas for Gating Free Scan Information

---

## The Prompt
What are the worst possible ways to gate or monetize the information from a free ConduitScore scan?
No filtering. No politeness. Generate the terrible ideas fast.

---

## The 20 Worst Ideas

1. **Show the score but scramble it randomly** — display a number like "47" but tell the user it's been "encrypted for privacy" and they need to pay to decrypt the real score.

2. **Blur every issue description after the first word** — user sees "Your robots.txt is blo—" and must pay to finish reading the sentence.

3. **Show all issues but randomize the severity labels** — critical issues might show as "info" and vice versa, so the user has no idea what's actually urgent.

4. **Display the score as a letter grade only** — never show the number, never explain what the grade means, just tell them they got a "C-" and leave them to guess why.

5. **Give the full report but delete it after 60 seconds** — a countdown timer burns the report, forcing them to upgrade to "save" it.

6. **Show all issues but hide which page each issue is on** — the user knows they have 12 issues but cannot locate a single one without paying.

7. **Require a phone number to view any results** — force immediate sales call scheduling before showing even the score.

8. **Show the full report but render it as a blurry JPEG** — like a screenshot printed at 15% opacity, technically "shown" but completely unreadable.

9. **Give all the code fixes but inject deliberate bugs into them** — the snippets look correct but won't work, user discovers on implementation that they need the "verified" paid version.

10. **Show only the worst issue and hide everything else** — "You have 1 critical problem (and 14 others). Pay to see the rest."

11. **Make every code fix expire** — snippets are valid for 24 hours. If they don't implement within a day, they must rescan and pay to regenerate.

12. **Replace all technical terms with synonyms** — instead of "robots.txt" say "crawler instruction manifest" so the user can't google the problem.

13. **Show the full report but require a $1 "verification fee" for each code snippet accessed** — micropayment per fix, à la iTunes 2003.

14. **Display only category scores, never item-level issues** — "Technical Health: 40/100" but no drill-down whatsoever.

15. **Show all issues but add a 30-second unskippable ad before each one loads** — twenty issues means ten minutes of ads.

16. **Gate the report behind a mandatory 10-question survey** — "Tell us about your business" before any data is shown.

17. **Show the score and issues, but present fix instructions as a riddle** — "What tag should go in your HTML head section? First one to guess gets the snippet."

18. **Lock all free reports behind a social share** — user must tweet their score publicly to unlock it, turning every low score into public humiliation.

19. **Show the full report only once, then replace all future views with "Report Archived"** — cannot revisit or share without upgrading.

20. **Require company email verification to see any results** — free Gmail addresses get nothing, forcing enterprise email addresses only, effectively killing all SMB access.
